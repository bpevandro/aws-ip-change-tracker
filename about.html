About

AWS IP Tracker is a serverless app that consists of two tools, Comparer and Find Service. These tools are intended to help AWS users to better visualize what has changed at AWS when it comes to IP addresses and also know whether or not a single (or multiple) IP(s) belongs to AWS, and, if so, what the service and region they're associated with are.

If you have ever asked one or all of the below question, this app is definitely going to help you:

  • Which IP addresses have been added to the Amazon organization since a given date?
  • Which IP addresses have been added to a specific service (e.g CloudFront) since a certain date?
  • Can two specific dates be compared to see whether a range of IPs was present at that point in time?
  • Do these IP addresses really belong to AWS so that I can allow them in my firewall?

How to use the tools

Both the tools are straight-forward. However, there are a couple of caveats that you should know, though. They are described below.


Comparer

This tool allows users to search for historical IP ranges from AWS and compare them with another one from a different date. Example, John accessed the tool and saw that there was a new date available in the datepickers, which means AWS has released new changes to its IP addresses, which in turn triggered this app to go and fetch the newly released 'ip-ranges.json' (https://ip-ranges.amazonaws.com/ip-ranges.json) and store it as an available date be searched through (date becomes available in the datepicker). Now John wants to know exactly which IP address has been added and its service and region. However, since he doesn't know the service which the newly added IP is associated with yet, he selects 'AMAZON' as the service, the last but one date available for "Date 1", and the last date available for "Date2. He can then take the IP address(es) showed on the result table and query it against the "Find Service" tool to know the service and region it's associated with.

The order of the dates you select dictates how the difference is calculated. The formula is:

DIFF = IPS in date2 - IPS in date1

Therefore, you should select the older date in the datepicker "Date 1" and the current(or more recent) one in the datepicker "Date 2", if you want to know if any IP has been added since Date 1.

However, if you want to know if an IP address has been removed, meaning that it was present in the older date, but not in the more recent one, select the more recent date in "Date 1", and the older one in "Date 2".
